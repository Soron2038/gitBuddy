design exploration · gitBuddy

Three directions, one decision.

Three intentional aesthetic stances for gitBuddy — each fully committed to a point of view, not a beige compromise. Click any tile to open the full mockup; both the tray popover and a snippet of the main window are shown side by side. Same placeholder data across all three, so only the design varies.

01 / Refined Native
Quiet, warm, distinctly Apple.

macOS Tahoe energy with translucent glass, soft sage as the accent, and Fraunces giving headings an editorial weight. Calm at a glance, beautiful close-up.

02 / Terminal Brutalist
Built for the keyboard.

Monospace-only, deep black, phosphor amber. Dense, tabular, ASCII separators, rate limits visible. For people who live in tmux and resent rounded corners.

JetBrains Mono only amber on black no radius dense grid

03 / Buddy (Warm)
A buddy with character.

A small sprout mascot, Gambarino's italic flourishes, terracotta and sage on cream. Friendly without being a toy — leans into the name without ever winking.

How to view: open design-mockups/index.html in your browser (Cmd+Double-click the file in Finder, or drag it onto Chrome/Safari). Each tile links to a full-size mockup. Resize the window to see the responsive collapse.

What to look at: typography choices, color temperature, density, the popover/menu-bar metaphor (top-right of each mockup), and the main-window layout below. Same data — anthropics/claude-code, mpsd/api-gateway, gitlab-org/gitlab, forgejo/runner, etc. — so differences are purely aesthetic.

Note on mixing: these are stances, not religions. After you pick, we can borrow elements — e.g. the warm typography from 03 with the data density of 02.
